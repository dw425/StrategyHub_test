Classification Metrics

Metrics for categorical predictions

Classification
Accuracy
(TP + TN) / (TP + TN + FP + FN)
Proportion of correct predictions. Simple but misleading for imbalanced classes.
Use when: Classes are balanced
Classification
Precision
TP / (TP + FP)
Of predicted positives, how many are actually positive? Minimize false positives.
Use when: False positives are costly (spam detection)
Classification
Recall (Sensitivity)
TP / (TP + FN)
Of actual positives, how many did we catch? Minimize false negatives.
Use when: False negatives are costly (fraud, disease)
Classification
F1 Score
2 √ó (Precision √ó Recall) / (Precision + Recall)
Harmonic mean of precision and recall. Balances both concerns.
Use when: Need balance, imbalanced classes
Classification
AUC-ROC
Area under ROC curve (TPR vs FPR)
Model's ability to distinguish classes across all thresholds. Threshold-independent.
Use when: Comparing models, threshold not fixed

Penalizes confident wrong predictions. Measures probability calibration.

Validation Strategies

Proper train/test splitting

‚úÇÔ∏è
Hold-Out Split
Simple train/test split (e.g., 80/20). Fast but high variance with small data.
üîÑ
K-Fold Cross-Validation
Split into K folds, train on K-1, test on 1. Repeat K times. More robust estimate.
‚öñÔ∏è
Stratified K-Fold
K-Fold preserving class distribution in each fold. Essential for imbalanced data.
üìÖ
Time Series Split
Train on past, test on future. Never leak future data into training.
üë•
Group K-Fold
Keep groups (e.g., users) together in same fold. Prevent data leakage.
üé≤
Nested Cross-Validation
Outer CV for evaluation, inner CV for hyperparameter tuning. Unbiased estimate.

Bias Detection

Fairness and bias considerations

Calibration

Best Practices

Model evaluation guidelines

‚úì Do's
  • Choose metrics aligned with business goals
  • Use cross-validation for robust estimates
  • Report confidence intervals, not just point estimates
  • Evaluate on multiple metrics
  • Test for bias across protected groups
  • Compare to meaningful baselines
‚úó Don'ts
  • Don't use accuracy for imbalanced classes
  • Never evaluate on training data
  • Avoid data leakage in validation
  • Don't ignore calibration for probabilities
  • Never skip fairness evaluation
  • Don't optimize for test set (use holdout)
