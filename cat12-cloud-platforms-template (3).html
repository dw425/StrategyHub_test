Cloud Architecture Patterns

Reference architectures for common cloud scenarios

Multi-Region Active-Active

Use for: Global apps, 99.99%+ SLA, disaster recovery
Latency: <50ms global
Failover: Automatic DNS-based
Data Sync: Async replication
Cost: 2x infrastructure

Microservices on Kubernetes

Use for: Large teams, complex domains, independent scaling
Scaling: Per-service HPA
Deploys: Independent CI/CD
Comms: gRPC + REST
Data: Database per service

Serverless Event-Driven

Use for: Variable workloads, event processing, APIs, cost optimization
Scale: 0 to millions auto
Pricing: Pay per execution
Cold Start: 100ms-2s
Max Duration: 15 min

Hybrid Cloud Architecture

Use for: Legacy integration, data sovereignty, gradual migration
Connect: VPN or Direct Connect
Identity: Federated SSO
Mgmt: Azure Arc / Anthos
Latency: 1-5ms private link

Data Lakehouse Architecture

Use for: Analytics, ML pipelines, unified data platform
Format: Delta Lake / Iceberg
ACID: Yes, transactional
Schema: Evolution supported
Cost: 70% less than DW

Event-Driven Architecture

Use for: Decoupled systems, real-time processing, async workflows
Coupling: Loose, async
Throughput: Millions/sec
Replay: Full history
Ordering: Per-partition

Migration Strategy Selection (6 R's)

Rehost
Lift & Shift
Move as-is to cloud VMs. Fastest path, minimal changes.
When: Time-critical, legacy apps, quick wins
Replatform
Lift & Reshape
Minor optimizations like managed DBs, containers.
When: Quick cloud benefits without refactoring
Refactor
Re-architect
Redesign for cloud-native. Serverless, microservices.
When: Strategic apps, long-term investment
Repurchase
Replace with SaaS
Move to commercial SaaS (Salesforce, Workday).
When: Commodity apps, reduce maintenance
Retain
Keep On-Premises
Stay on-prem for now. Revisit later.
When: Compliance, latency, recent investment
Retire
Decommission
Turn off unused or redundant applications.
When: No business value, duplicate systems
