Network Architecture Patterns

Modern approaches to network design

Hub & Spoke Topology

Use for: Centralized management, branch offices, cost optimization
Pros: Simple, centralized control
Cons: Single point of failure
Latency: All traffic through hub

Full Mesh Topology

Use for: High availability, low latency, critical infrastructure
Pros: Maximum redundancy
Cons: Expensive, complex
Links: n(n-1)/2 connections

Spine-Leaf Architecture

Use for: Data centers, east-west traffic, predictable latency
Hops: Always 2 (leaf→spine→leaf)
Scale: Add leaves horizontally
Used by: AWS, Azure, Google DCs

SD-WAN Architecture

Use for: Multi-site WAN, cloud access, hybrid connectivity
Benefit: App-aware routing
Transports: MPLS + Internet + LTE
Vendors: Cisco Viptela, VMware VeloCloud

Zero Trust Network

Use for: Modern security, remote workforce, micro-segmentation
Principle: Least privilege access
Auth: Every request verified
Vendors: Zscaler, Cloudflare, Palo Alto
