Statistical Methods

Foundation techniques for data analysis, inference, and modeling

Category Methods When to Use Example
üìà Descriptive Mean, Median, Mode, Std Dev, Variance, Percentiles, IQR, Skewness, Kurtosis Summarize data distribution, identify outliers, understand shape df.describe() ‚Üí Quick profile of all numeric columns
üéØ Hypothesis Testing t-test (1-sample, 2-sample, paired), Chi-square, Mann-Whitney U, Wilcoxon, Kolmogorov-Smirnov Compare groups, test assumptions, validate differences are significant Is treatment group conversion rate > control? ‚Üí 2-sample t-test
üìä ANOVA One-way ANOVA, Two-way ANOVA, MANOVA, ANCOVA, Repeated Measures, Post-hoc (Tukey, Bonferroni) Compare 3+ groups, analyze variance across factors Do 4 marketing campaigns have different ROI? ‚Üí One-way ANOVA
üîó Correlation Pearson (linear), Spearman (rank), Kendall (ordinal), Point-biserial, Partial correlation Measure relationships between variables, detect multicollinearity Is ad spend correlated with revenue? ‚Üí Pearson r = 0.85
üìâ Regression Linear, Multiple, Polynomial, Logistic, Poisson, Ridge, Lasso, Elastic Net, Quantile Predict outcomes, understand drivers, model relationships Predict house price from features ‚Üí Multiple Linear Regression
‚è±Ô∏è Time Series ARIMA, SARIMA, Exponential Smoothing, Holt-Winters, STL Decomposition, Granger Causality Forecast future values, detect seasonality, analyze trends Forecast next quarter sales ‚Üí SARIMA with seasonal adjustment
üé≤ Probability Bayes' Theorem, Conditional Probability, Distributions (Normal, Binomial, Poisson, Exponential) Model uncertainty, calculate likelihoods, Bayesian inference Given positive test, P(disease)? ‚Üí Bayes' Theorem
üî¨ Non-Parametric Kruskal-Wallis, Friedman, Sign Test, Permutation Tests, Bootstrap, Kernel Density When normality assumptions violated, small samples, ordinal data Compare medians with skewed data ‚Üí Kruskal-Wallis test
üìê P-Value
Probability of observing results if null hypothesis is true. p < 0.05 typically considered significant.
üìä Confidence Interval
Range where true parameter lies with X% confidence. 95% CI means 95 of 100 samples contain true value.
üìà Effect Size
Magnitude of difference (Cohen's d, R¬≤). Large samples can show significance for tiny effects.
‚ö° Statistical Power
Probability of detecting true effect. 80% power standard. Increases with sample size.

Feature Engineering

Transform raw data into predictive signals ‚Äî where domain expertise meets data science

Technique Description Methods Code Example
üî¢ Numeric Scaling Normalize range for algorithms sensitive to scale (SVM, kNN, Neural Nets) StandardScaler (z-score), MinMaxScaler (0-1), RobustScaler scaler.fit_transform(X[['age','income']])
üè∑Ô∏è Categorical Encoding Convert categories to numbers while preserving meaning One-hot, Label, Target, Frequency, Binary, Ordinal pd.get_dummies(df, columns=['city'])
üìÖ Temporal Features Extract time-based signals from datetime columns Day/Month/Year, DayOfWeek, Hour, IsWeekend, Quarter, Cyclical df['dow'] = df['date'].dt.dayofweek
‚è≥ Lag Features Use historical values as predictors (time series) Lag (t-1, t-7), Rolling mean/std, Expanding stats, Diff df['sales_lag7'] = df['sales'].shift(7)
üìù Text Features Convert text into numeric representations TF-IDF, Word2Vec, BERT embeddings, N-grams, Sentiment TfidfVectorizer().fit_transform(texts)
üîÄ Interactions Combine features to capture relationships Products, Ratios, Differences, Polynomial combinations df['price_per_sqft'] = df['price']/df['sqft']
üìâ Binning Convert continuous to categorical (reduce noise, handle outliers) Equal-width, Equal-frequency, Custom bins, K-means bins pd.cut(df['age'], bins=[0,18,35,50,100])
üìä Aggregations Create features by grouping and summarizing GroupBy stats (mean, count, max), Window functions df.groupby('user')['orders'].transform('mean')
üîç Missing Values Handle nulls with imputation or indicators Mean/Median/Mode fill, KNN impute, Indicator columns SimpleImputer(strategy='median').fit_transform(X)
üéØ Dim Reduction Reduce features while preserving information PCA, t-SNE, UMAP, Feature selection (RFE, LASSO) PCA(n_components=10).fit_transform(X)

Data Science Simplified

Start making an impact today ‚Äî no PhD required

üéØ What Data Science Really Is
Data science is simply using data to answer questions and make better decisions. You don't need complex algorithms to start ‚Äî most business value comes from basic analysis done well.
üí° The 80/20 Rule of Data Science
80% of the value comes from simple techniques: descriptive stats, basic regression, and clear visualization. 20% of projects actually need advanced ML. Start simple.
‚úÖ You're Ready If You Can:
‚Ä¢ Ask a specific business question
‚Ä¢ Access relevant data (even Excel works)
‚Ä¢ Calculate averages and percentages
‚Ä¢ Create a chart that tells a story
‚ö° Start Today: 30-Minute Project

# 4. Find insight: "December is 3x higher"
# 5. Take action: Staff up for holidays!

Data Science Agent

AI-powered assistant for end-to-end data science workflows

üî¨
DataScienceAgent
ML Research Specialist
Automates the complete data science workflow from EDA through model deployment. Handles feature engineering, model selection, hyperparameter tuning, and generates comprehensive analysis reports.
  • Automated exploratory data analysis
  • Feature engineering suggestions
  • Multi-model comparison
  • Hyperparameter optimization
  • Model interpretation (SHAP)
Agent Definition
Analysis Task
ds_agent.py
# ds_agent.py - Data Science Agent
from crewai import Agent, Task, Crew

data_scientist = Agent(
    role="Senior Data Scientist",
    goal="Build high-performing predictive models",
    backstory="PhD in Statistics with 10 years ML experience",
    tools=[
        DataProfiler(),
        FeatureEngineer(),
        ModelSelector(),
        HyperparamTuner(),
        ModelInterpreter(),
    ]
)

modeling_task = Task(
    description="""
    Objective: Predict customer churn with >90% accuracy
    
    1. Profile dataset and identify quality issues
    2. Engineer features from transaction history
    3. Compare baseline models (LR, RF, XGB, NN)
    4. Tune top performer with Optuna
    5. Generate SHAP explanations for top drivers
    6. Create deployment-ready model package
    """,
    agent=data_scientist,
    expected_output="Production model with documentation"
)

# Execute workflow
crew = Crew(agents=[data_scientist], tasks=[modeling_task])
result = crew.kickoff()
