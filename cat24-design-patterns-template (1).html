Architectural Patterns

High-level structures for organizing software systems

Model-View-Controller (MVC)

Separates application into three interconnected components
Best For
Web applications, desktop apps with complex UIs
Frameworks
Ruby on Rails, ASP.NET MVC, Spring MVC, Django
Trade-offs
Clear separation but can lead to complex controllers

Model-View-ViewModel (MVVM)

Two-way data binding between View and ViewModel
Best For
Rich client applications, mobile apps
Frameworks
WPF, Xamarin, Angular, Vue.js, SwiftUI
Trade-offs
Great testability but data binding can be complex

Microservices Architecture

Decompose application into loosely coupled services
Best For
Large-scale applications, distributed teams
Key Tech
Kubernetes, Docker, Service Mesh, API Gateway
Trade-offs
Scalable but complex operations and debugging

Event-Driven Architecture

Asynchronous communication via events
Best For
Real-time systems, decoupled services
Key Tech
Kafka, RabbitMQ, AWS SNS/SQS, Redis Pub/Sub
Trade-offs
Highly decoupled but eventual consistency challenges

Command Query Responsibility Segregation

Separate read and write operations
Best For
High-read systems, complex domains
Key Tech
Event Sourcing, Read Replicas, Redis Cache
Trade-offs
Optimized reads/writes but increased complexity

Hexagonal Architecture (Ports & Adapters)

Isolate core logic from external dependencies
Best For
Testable, maintainable applications
Key Concept
Dependency inversion, interface-based design
Trade-offs
Clean architecture but more boilerplate code
