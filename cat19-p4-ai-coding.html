Overview

The AI-assisted development landscape

What Are AI Coding Tools?

AI coding tools are software that uses large language models to assist with programming tasks‚Äîfrom simple autocomplete to autonomous code generation. They understand code, natural language, and the relationship between them.

The goal: augment developer productivity by handling routine tasks, suggesting solutions, and enabling developers to work at higher levels of abstraction.

Tool Categories

  • IDE Assistants ‚Äî Cursor, Windsurf, VS Code + Extensions
  • Code Extensions ‚Äî GitHub Copilot, Cody, Tabnine, Amazon Q
  • CLI Tools ‚Äî Claude Code, Aider, GPT-Engineer
  • Autonomous Agents ‚Äî Devin, Factory, Cosine Genie
  • Code Review ‚Äî CodeRabbit, Graphite, What The Diff
  • Specialized ‚Äî Copilot Workspace, Replit Agent

The Paradigm Shift

We're moving through distinct phases: autocomplete (predict the next line), chat-assisted (ask questions about code), agentic (delegate multi-step tasks). The end state is AI that can take a feature request and deliver working, tested code‚Äîwith humans providing guidance and review. The fastest developers are already 2-3x more productive with AI tools. The gap between AI-augmented and traditional development is widening rapidly.

Prompting Strategies

Getting better results from AI coding tools

üí°

Prompts Are Programs

The quality of AI output directly reflects the quality of your prompt. Vague requests get vague results. Treat prompting as a skill to develop. The best developers are learning to write prompts as carefully as they write code.

üìã
Be Specific

Bad: "Add authentication"
Good: "Add JWT-based authentication with refresh tokens. Use the existing User model. Add login/logout/refresh endpoints to /api/auth/. Store refresh tokens in Redis."

üìÅ
Reference Context

Bad: "Fix the bug"
Good: "Look at @src/api/users.ts and @src/db/queries.ts. The getUser function returns null when user exists. Check the query condition."

üéØ
Specify Style

Bad: "Write a function"
Good: "Write a TypeScript function with JSDoc comments. Use async/await, not callbacks. Follow the patterns in @src/utils/. Include error handling."

üîÑ
Iterate Progressively

Bad: "Build the entire payment system"
Good: Start with "Create the payment model" ‚Üí "Add the Stripe integration" ‚Üí "Write the checkout endpoint" ‚Üí "Add tests"

Example .cursorrules file
# Project Context
This is a Next.js 14 application with TypeScript.
We use Tailwind CSS for styling and Prisma for database.
The app follows a modular architecture in src/modules/.

# Code Style
- Use functional components with hooks
- Prefer named exports over default exports
- Use async/await, never callbacks
- Add JSDoc comments for public functions
- Use Zod for runtime validation

# Patterns
- API routes go in src/app/api/
- Database queries in src/db/queries/
- Shared components in src/components/ui/
- Use the Result type for error handling

# Testing
- Write tests with Vitest
- Mock external services
- Test both success and error paths

# Don'ts
- Don't use any type
- Don't hardcode credentials
- Don't skip error handling
- Don't modify migrations directly

Best Practices

Getting the most from AI coding tools

1

Write Clear Prompts

Be specific about requirements. Include context. Mention constraints. The quality of output mirrors the quality of input.

2

Review Everything

AI makes mistakes. Security issues, subtle bugs, poor patterns. Treat AI code like code from a junior developer‚Äîreview carefully.

3

Use Project Context Files

.cursorrules, CLAUDE.md, AGENTS.md‚Äîgive AI context about your project's patterns, conventions, and architecture.

4

Iterate in Small Steps

Don't ask for complete features at once. Build incrementally. Review each step. Correct course early.

5

Leverage Codebase Context

Use @codebase, @file references. Help AI understand your patterns. More context = better output.

6

Know When Not to Use AI

Novel architectures, security-critical code, performance-sensitive paths‚Äîsometimes manual is better.

7

Keep Tests Running

AI can introduce bugs. Continuous test execution catches issues early. AI + TDD = powerful combination.

8

Learn the Shortcuts

Cmd+K, Cmd+L, tab, reject. Speed comes from muscle memory. Invest time in learning your tool's UX.

# CodeAssistant Agent
from crewai import Agent, Task, Crew
from langchain_anthropic import ChatAnthropic
from tools import (
    FileReader, CodeSearcher, TestRunner,
    GitOperations, LintChecker
)

llm = ChatAnthropic(model="claude-sonnet-4-20250514")

code_assistant = Agent(
    role="Senior Software Engineer",
    goal="""Implement features and fix bugs 
    with high-quality, tested code""",
    backstory="""Expert developer who writes 
    clean, maintainable code. Follows best 
    practices, writes tests, and documents 
    changes thoroughly.""",
    tools=[
        FileReader(),
        CodeSearcher(),
        TestRunner(),
        GitOperations(),
        LintChecker()
    ],
    llm=llm,
    verbose=True
)

async def implement_feature(
    description: str,
    context_files: list[str]
) -> dict:
    """Implement a feature with AI assistance."""
    
    task = Task(
        description=f"""
        Implement the following feature:
        {description}
        
        Context files: {context_files}
        
        Steps:
        1. Understand existing code patterns
        2. Plan implementation approach
        3. Write code following project style
        4. Add appropriate tests
        5. Run linter and fix issues
        6. Document changes
        """,
        agent=code_assistant,
        expected_output="Implemented feature with tests"
    )
    
    crew = Crew(agents=[code_assistant], tasks=[task])
    result = await crew.kickoff_async()
    
    return {
        "files_changed": result.files,
        "tests_added": result.tests,
        "documentation": result.docs
    }
