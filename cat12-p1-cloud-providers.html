Pricing Models

Understanding how cloud providers charge

‚è±Ô∏è
On-Demand
Pay for compute capacity by the second or hour with no long-term commitments. Maximum flexibility, premium pricing.
  • AWS: EC2 On-Demand
  • Azure: Pay-As-You-Go VMs
  • GCP: On-Demand pricing
üìÖ
Reserved / Committed
1-3 year commitments for significant discounts (30-72%). Best for steady-state workloads.
  • AWS: Reserved Instances, Savings Plans
  • Azure: Reserved VM Instances
  • GCP: Committed Use Discounts
üé≤
Spot / Preemptible
Spare capacity at 60-90% discount. Can be interrupted with notice. Great for batch workloads.
  • AWS: Spot Instances
  • Azure: Spot VMs
  • GCP: Preemptible / Spot VMs

AWS Unique Pricing

Savings Plans: Flexible commitment covering EC2, Fargate, and Lambda usage with up to 72% savings.

Spot Fleet: Automated spot instance management across multiple pools for maximum availability.

Azure Unique Pricing

Hybrid Benefit: Use existing Windows Server/SQL licenses in cloud for up to 40% savings.

Dev/Test Pricing: Reduced rates for development and testing workloads.

GCP Unique Pricing

Sustained Use Discounts: Automatic discounts (up to 30%) just for running VMs 25%+ of month‚Äîno commitment needed. Per-Second Billing: All compute billed per-second with 1-minute minimum.

Provider Selection Guide

When to choose each cloud provider

AWS
Choose AWS When...
  • You need the broadest service catalog
  • Global scale with maximum coverage
  • Serverless/event-driven is primary pattern
  • Extensive marketplace integrations needed
  • Team has existing AWS expertise
  • Complex multi-service architectures
Azure
Choose Azure When...
  • You're a Microsoft shop (Windows, .NET, M365)
  • Hybrid cloud with on-prem is essential
  • Enterprise identity (Entra ID) integration
  • Government/compliance requirements
  • Leverage existing Microsoft licensing
  • Power Platform is part of strategy
GCP
Choose GCP When...
  • Data analytics/warehousing (BigQuery) core
  • Best-in-class Kubernetes (GKE) needed
  • AI/ML workloads are significant
  • Transparent, predictable pricing matters
  • Low-latency global networking critical
  • Open-source technologies preferred

Multi-Cloud Considerations

Many enterprises adopt multi-cloud for: Best-of-breed (use each provider's strengths), Avoid lock-in, Compliance (data residency), M&A (inherited infrastructure), or Resilience (DR across providers).

‚ö†Ô∏è Warning: Multi-cloud adds complexity in networking, identity, operations, and skills. Only adopt if clear business drivers justify the overhead. "Single cloud, multi-region" often provides better resilience with less complexity.

    agent=cloud_advisor
