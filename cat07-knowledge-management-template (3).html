RAG & Semantic Search

Retrieval Augmented Generation for accurate, grounded AI responses

Retrieval Augmented Generation (RAG) combines the power of large language models with your organization's proprietary knowledge. Instead of relying solely on the LLM's training data, RAG retrieves relevant documents from your knowledge base and uses them to generate accurate, contextual responses grounded in your actual content.

The RAG pipeline transforms documents into vector embeddings, stores them in a vector database, and retrieves semantically similar content when users ask questions. This approach dramatically reduces hallucinations while keeping responses current with your latest documentation.

  • Knowledge graph entity extraction
  • Semantic search optimization

# Knowledge Management Crew - Blueprint cOS
from crewai import Agent, Task, Crew, Process
from langchain_anthropic import ChatAnthropic

# Initialize LLM
llm = ChatAnthropic(model="claude-sonnet-4-20250514")

# Define Knowledge Agents
content_curator = Agent(
    role="Content Curator",
    goal="Organize and tag content effectively",
    backstory="""Expert librarian who understands 
    information architecture and content organization.""",
    llm=llm,
    tools=[notion_api, confluence_api, tagging_tool]
)

knowledge_extractor = Agent(
    role="Knowledge Extractor",
    goal="Extract entities and relationships from content",
    backstory="""NLP specialist who identifies key concepts,
    facts, and connections within documents.""",
    llm=llm,
    tools=[ner_tool, relation_extractor, neo4j_api]
)

search_optimizer = Agent(
    role="Search Optimizer",
    goal="Maximize content discoverability",
    backstory="""Search expert who understands semantic
    search, embeddings, and query optimization.""",
    llm=llm,
    tools=[pinecone_api, embedding_model, analytics_api]
)

content_auditor = Agent(
    role="Content Auditor",
    goal="Maintain content quality and freshness",
    backstory="""Quality analyst who identifies stale,
    duplicate, or inconsistent content.""",
    llm=llm,
    tools=[diff_tool, freshness_checker, duplicate_finder]
)

# Create Knowledge Crew
knowledge_crew = Crew(
    agents=[content_curator, knowledge_extractor, 
            search_optimizer, content_auditor],
    process=Process.sequential,
    verbose=True
)

async def process_new_document(doc_id: str):
    """Process a new document through the knowledge pipeline."""
    tasks = [
        Task(
            description="Extract key entities and facts",
            agent=knowledge_extractor,
            expected_output="Entities and relationships"
        ),
        Task(
            description="Classify and tag the document",
            agent=content_curator,
            expected_output="Tags and category assignments"
        ),
        Task(
            description="Generate embeddings and index",
            agent=search_optimizer,
            expected_output="Indexed in vector store"
        ),
        Task(
            description="Check for duplicates and conflicts",
            agent=content_auditor,
            expected_output="Quality assessment report"
        )
    ]
    return await knowledge_crew.kickoff_async(tasks=tasks)
