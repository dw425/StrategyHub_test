Prompt Engineering

Master techniques for getting the best results from LLMs

Technique Description When to Use Example Pattern
üìù Zero-Shot Direct instruction without examples; relies on model's trained knowledge Simple tasks, well-defined outputs, testing baselines "Classify this text as spam or not spam: [text]"
üìã Few-Shot Provide 2-5 examples demonstrating desired format and behavior Custom formats, domain-specific tasks, improving accuracy "Ex: 'Love it!' ‚Üí Positive\nEx: 'Terrible' ‚Üí Negative\nNow classify: [text]"
üîó Chain-of-Thought Encourage step-by-step reasoning before final answer Math, logic, complex reasoning, debugging "Let's think step by step..." or "First analyze X, then consider Y..."
üé≠ Role/Persona Assign expertise or character to shape response style and depth Expert-level answers, specific perspectives, consistent tone "You are a senior backend engineer with 15 years of experience..."
üå≥ Tree of Thoughts Explore multiple reasoning paths, evaluate each, select best Planning, creative problems, strategic decisions "Consider 3 approaches, evaluate pros/cons, recommend best..."
üîÑ Self-Consistency Generate multiple answers and select most common/consistent Critical decisions, reducing hallucinations, verification "Solve 5 ways, then pick the answer that appears most often"
üîç ReAct Reason + Act: interleave reasoning with tool use actions Agents, tool use, multi-step tasks, research "Thought: I need to... Action: search() Observation: ... Thought: Now I can..."
üìê Optimal Prompt Structure
# CONTEXT (who/what)
You are a [role] helping with [task].
# INSTRUCTIONS (how)
Follow these steps: 1. ... 2. ... 3. ...
# EXAMPLES (optional)
Input: X ‚Üí Output: Y
# INPUT (what to process)
Here is the content: [content]
# OUTPUT FORMAT (constraints)
Respond in JSON with keys: ...
üí° Pro Tips
‚úì Be specific ‚Äî "Write 3 paragraphs" not "Write something"
‚úì Use delimiters ‚Äî Triple quotes, XML tags, or markdown for structure
‚úì Specify format ‚Äî "Return as JSON" or "Use bullet points"
‚úì Give examples ‚Äî Show don't tell for complex outputs
‚úì Iterate ‚Äî Test, evaluate, refine prompts systematically
‚úó Avoid vague instructions ‚Äî "Make it better" won't help
‚úó Don't overload ‚Äî One task per prompt is clearer
‚úó Skip negative framing ‚Äî "Do X" not "Don't do Y"

‚úÇÔ∏è Chunking Strategies
Fixed-size: Simple, consistent chunks (512 tokens)
Recursive: Split on paragraphs, then sentences
Semantic: Group by topic similarity
Overlap: 10-20% overlap prevents cut-off context
üí° Smaller chunks = more precise, larger = more context
üîç Retrieval Methods
Semantic: Cosine similarity on embeddings
Keyword (BM25): Traditional text matching
Hybrid: Combine semantic + keyword (RRF)
Reranking: Re-score with cross-encoder
üí° Hybrid + Rerank = best accuracy
üöÄ Advanced RAG Patterns
HyDE: Generate hypothetical doc first
Multi-query: Expand query for coverage
Self-RAG: Retrieve only when needed
Graph RAG: Knowledge graph + vectors
üí° Start simple, add complexity as needed

Fine-Tuning Approaches

Customize models for specific domains, tasks, and behaviors

Method Description Params Cost Data Needed Best For
Full Fine-Tuning Update all model weights on task-specific data 100% $$$$ 10K-100K Maximum customization, new domains
LoRA Low-rank matrices added to frozen base model 0.1-1% $ 1K-10K Style adaptation, task specialization
QLoRA LoRA with 4-bit quantization for memory efficiency 0.1-1% $ 1K-10K Consumer GPUs, limited memory
RLHF Train reward model, then PPO to align with preferences Varies $$$ Preference pairs Behavior alignment, safety, helpfulness
DPO Direct preference optimization without reward model Varies $$ Preference pairs Simpler than RLHF, similar results
Prompt Tuning Learn soft prompts while freezing entire model 0.01% $ 100-1K Minimal changes, multi-tenant
ü§î When to Fine-Tune vs RAG vs Prompting
Fine-tune: Need consistent style/tone, specialized behavior, faster inference
RAG: Need factual grounding, up-to-date info, source attribution
Prompting: Quick iteration, no training data, general tasks
üí° Often combine: Fine-tune for style + RAG for facts

‚òë Clean and deduplicate data
‚òë Balance positive and negative examples
‚òë Hold out 10-20% for validation
‚òë Review for PII and sensitive content

from crewai import Agent, Task, Crew
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

genai_architect = Agent(
    role="GenAI Architect",
    goal="Build production LLM applications",
    backstory="Expert in RAG, prompting, and LLM deployment",
    tools=[
        DocumentLoader(),
        EmbeddingGenerator(),
        VectorStoreManager(),
        PromptOptimizer(),
        ResponseEvaluator(),
    ]
)

rag_task = Task(
    description="""
    Build a RAG system for customer support:
    1. Ingest knowledge base documents
    2. Chunk and embed with optimal settings
    3. Configure vector store with metadata
    4. Design retrieval prompts
    5. Implement response generation
    6. Add guardrails and evaluation
    """,
    agent=genai_architect,
    expected_output="Production RAG with 90%+ relevance"
)

# Execute pipeline build
crew = Crew(agents=[genai_architect], tasks=[rag_task])
result = crew.kickoff()
